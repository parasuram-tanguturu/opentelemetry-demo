# Project Brief: OpenTelemetry Demo

## Overview
The OpenTelemetry Demo (also known as the "Astronomy Shop") is a microservice-based distributed system designed to demonstrate OpenTelemetry instrumentation and observability in a realistic, near-production environment.

## Core Purpose
This project serves three primary goals:

1. **Demonstration Platform**: Provide a realistic example of a distributed system that showcases OpenTelemetry instrumentation and observability capabilities
2. **Vendor Extension Base**: Build a foundation for vendors, tooling authors, and others to extend and demonstrate their OpenTelemetry integrations
3. **Testing Ground**: Create a living example for OpenTelemetry contributors to test new versions of the API, SDK, and other components

## Project Scope
- Multi-language microservices architecture (Java, Go, .NET, Python, Node.js, Rust, C++, PHP, Ruby, Elixir, Kotlin, React Native)
- Full observability stack integration (traces, metrics, logs)
- Real-world e-commerce use case (Astronomy Shop)
- Docker and Kubernetes deployment options
- Comprehensive testing infrastructure
- Documentation and contribution guidelines

## Key Characteristics
- **Realistic**: Simulates real-world distributed system patterns
- **Educational**: Demonstrates best practices for OpenTelemetry instrumentation
- **Extensible**: Designed to be forked and extended by vendors
- **Community-Driven**: Maintained by OpenTelemetry community with regular SIG calls

## Success Criteria
- Successfully demonstrates OpenTelemetry capabilities across multiple languages
- Provides clear examples for developers learning OpenTelemetry
- Serves as reliable testbed for OpenTelemetry development
- Maintains compatibility with latest OpenTelemetry specifications
